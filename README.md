## 先搞清楚：搬瓦工连接不上，到底有哪几种情况

搬瓦工连接不上，表现各不相同，根本原因也不一样。大致分四种：

1. **IP 被封（国内封，国外可访问）** — 最常见，也最头疼
2. **TCP 端口阻断（能 ping 通，SSH 却死活连不上）** — 比 IP 被封更隐蔽
3. **VPS 没正常运行（宕机 / 流量超限 / 账单异常）** — 可以自己在后台处理
4. **本地网络或线路波动** — 通常等一等就好，或切换宽带类型

---

## 第一步：先判断是哪种情况

搞清楚是哪种情况，才能不走弯路。

**方法一：用 ping 初步判断**

打开命令行，输入 `ping 你的VPS_IP`。

- **超时（全部 timeout）** → 可能是 IP 被封，或 VPS 没开机
- **能 ping 通，但 SSH 连不上** → 大概率是 TCP 端口阻断

只靠 ping 判断现在已经不够准了。有一种情况是 ICMP 包放行了，但 TCP 包被拦，这叫 TCP 阻断，Ping 正常但 SSH 就是连不上。

**方法二：区分国内/国外可用性**

建议去找多地 ping 测试工具，输入你的 VPS IP，看看国内节点和国外节点的测试结果：

- 国内超时、国外正常 → **IP 被封**
- 国内外都超时 → VPS 可能没开机，或防火墙把 ICMP 全封了
- 都能 ping，但 SSH 不行 → **TCP 端口阻断**

**方法三：直接进 KiwiVM 后台看**

登录搬瓦工的 KiwiVM 控制面板，这是最直接的方式。KiwiVM 面板里有官方提供的 IP 检测工具（`blacklistcheck`），点 Test Main IP 就能看结果。另外面板里也能看到 VPS 当前状态——是 running、stopped 还是被 suspended。

## 情况一：IP 被封（国内访问不了，国外正常）

这是用搬瓦工最怕遇到的情况。

**判断标准**：国内无法访问，国外正常，但 KiwiVM 后台还能打开。

IP 被封之后，最直接的解法是**更换 IP**。搬瓦工支持付费换 IP，费用是 $8.79 一次，换完之后会随机分配新 IP，并保证新 IP 在全球可达（通过官方工具验证）。如果你的套餐支持迁移机房，有时候迁机房也会顺带换 IP，这是另一个思路。

说实话，IP 被封更多是使用方式导致的。正常建站、学 Linux、跑应用，遇到 IP 被封的概率不高。如果频繁遇到，可能需要重新审视一下自己的使用方式。

另外，**选更高端的线路套餐能明显降低 IP 被封的概率**。普通 CN2 GT 线路和 CN2 GIA 线路的抗封性差别还是比较明显的，后面套餐对比的部分会聊到。

## 情况二：TCP 端口阻断（能 ping，SSH 连不上）

这个比 IP 直接被封更难判断，但遇到的人也不少。

**核心原因**：ICMP 协议（ping 用的）没被拦，但 TCP 协议（SSH 用的）被拦了。所以 ping 通，SSH 超时。

**解决方法**：

1. **换 SSH 端口** — 有时候是特定端口被封，换一个端口就能连上。在 KiwiVM 面板里可以操作。
2. **迁移机房** — 换个机房，IP 随之变化，往往能解决端口阻断问题。CN2 GIA-E 套餐支持在十几个机房之间自由切换，这是这套餐的一大实用价值。
3. **更换 IP** — 如果换端口和迁机房都不管用，只能付费换 IP 了。

## 情况三：VPS 没有正常运行

这种情况处理起来相对简单，大多数情况下不需要联系客服。

**VPS 宕机 / 没正常开机**

进 KiwiVM 后台，先看 VPS 状态。如果显示 stopped 或者 suspended，依次点 force stop → start（注意顺序），重启一下通常就好了。如果 running 状态但连不上，进 VNC 页面看看系统内部是否正常。

**月流量超限**

搬瓦工每个套餐都有月流量上限。一旦用超，系统会自动暂停 VPS，表现就是突然连不上、网站打不开，就像服务器消失了。

流量每月按订购日期重置，不是按自然月。比如你 15 号购买的，那每月 15 号重置。进 KiwiVM 能看到剩余流量，快超的时候提前留意一下。

**账单信息异常**

有时候账单地址格式不对、电话号码格式错误，系统会暂停服务。登录搬瓦工账户后台，看有没有相关警告提示。修改账单信息后一般会自动恢复，如果没恢复，提工单联系客服处理，通常 1-3 个工作日内解决。

## 情况四：本地网络 / 线路波动

这种情况下，VPS 本身没问题，是你本地和机房之间的网络路径出了问题。

晚高峰（晚 8 点到 11 点）丢包率高是老问题。普通线路的出口在晚高峰会比较拥挤，有时候丢包能到 20-30%，体感就是连不上或者极度卡顿。

**临时解决**：等一等，或者把手机热点开起来换个网络环境试试，有时候切换运营商（比如从电信切到移动）就改善了。

**根本解决**：选 CN2 GIA 线路的套餐。这条线路走的是中国电信的精品骨干网，晚高峰拥挤程度远比普通线路低，实际用下来断线和连不上的频率明显更少。

## 自查清单：遇到搬瓦工连接不上，按这个顺序排查

1. 先检查搬瓦工官方服务状态（bwhstatus.com），确认不是机房全线故障
2. 进 KiwiVM 看 VPS 是否在运行状态，查看流量余量和账单状态
3. 测试国内外 ping：区分 IP 被封 vs 本地线路问题
4. 如果 ping 通但 SSH 连不上，考虑 TCP 端口阻断，尝试换端口或迁机房
5. 如果是 IP 被封，决定付费换 IP 或迁机房

大多数情况，走到第 3 步就能定位问题了。
